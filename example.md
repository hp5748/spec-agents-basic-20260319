## 关键文件索引

| 文件路径 | 作用 |
|---------|------|
| `skills/sqlite-query-skill/SKILL.md` | 技能元数据（keywords, intents, description） |
| `skills/sqlite-query-skill/scripts/executor.py` | 数据库查询执行逻辑 |
| `src/skill_loader.py` | 加载 Skill 资源（SKILL.md 解析） |
| `src/skill_executor.py` | 动态加载并执行 executor.py |
| `src/intent/recognizer.py` | 意图识别（关键词匹配） |
| `src/agent/stream_agent.py` | 流式 Agent 主逻辑 |
| `src/web/routes/chat.py` | Web API 路由 |

## 数据流总结

1. **用户输入** → Web API / CLI
2. **StreamAgent** 接收请求
3. **IntentRecognizer** 基于关键词匹配识别意图
4. **SkillExecutor** 动态加载对应的 executor.py
5. **executor.py** 执行具体业务逻辑（参数提取 → SQL 构建 → 数据库查询 → 结果格式化）
6. **返回 SkillResult** 给 StreamAgent
7. **StreamAgent** 流式返回响应给用户

## Skill 开发规范

基于 `sqlite-query-skill` 的实现，一个标准 Skill 需要包含：

### 1. SKILL.md（必需）

```yaml
---
name: skill-name
description: 技能描述
version: 1.0.0
priority: 10
intents:
  - intent_name
keywords:
  - 关键词1
  - 关键词2
examples:
  - "示例输入1"
  - "示例输入2"
---

# 技能详细说明
...
```

### 2. scripts/executor.py（必需）

```python
def execute(context: Dict[str, Any], input_data: Dict[str, Any]) -> Dict[str, Any]:
    """
    执行函数 - 必须实现

    Args:
        context: 执行上下文 (session_id 等)
        input_data: 输入数据 (user_input, intent 等)

    Returns:
        Dict: 必须包含 success, response, data 字段
    """
    return {
        "success": True,
        "response": "响应文本",
        "data": {}  # 结构化数据
    }
```

### 3. 可选目录

- `templates/` - 模板文件
- `examples/` - 示例文件
- `references/` - 参考文档
