# 团队通信协议

## 消息格式

所有消息使用 Markdown 格式，包含以下元信息：

```markdown
# 消息标题

## 元信息
- **发送者**: [角色]
- **接收者**: [角色]
- **时间**: [时间戳]
- **类型**: [request/response/notification]
- **优先级**: [high/medium/low]

## 内容
[消息正文]
```

## 通信方式

1. **同步通信**: 使用 Agent 工具直接启动子代理
2. **异步通信**: 通过 inbox/outbox 目录传递消息文件

## 目录说明

| 目录 | 用途 |
|------|------|
| inbox/[角色]/ | 该角色的收件箱 |
| outbox/[角色]/ | 该角色的发件箱 |
| knowledge/ | 共享知识库 |
| knowledge/decisions/ | 架构决策记录 |
| knowledge/lessons/ | 经验教训 |

## 角色通信矩阵

| 发送者 | 接收者 | 通信场景 |
|--------|--------|----------|
| PM | 项目经理 | PRD 完成后分发 |
| 项目经理 | 架构师 | 架构设计任务 |
| 项目经理 | 开发者 | 编码任务 |
| 项目经理 | 测试 | 测试任务 |
| 开发者 | 审查者 | 代码审查请求 |
| 审查者 | 开发者 | 审查反馈 |
| 测试 | 开发者 | Bug 报告 |

## 消息优先级

- **high**: 阻塞性问题、紧急 Bug
- **medium**: 常规任务、功能开发
- **low**: 优化建议、文档更新
