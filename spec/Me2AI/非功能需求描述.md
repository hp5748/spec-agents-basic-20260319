# 非功能需求描述

## 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| API响应时间 | < 5秒 | 含LLM调用 |
| 流式首字延迟 | < 1秒 | SSE首字节 |
| 并发支持 | 10 QPS | Demo级别 |
| 技能加载时间 | < 2秒 | 启动时 |

### 前端性能指标

| 指标 | 要求 | 说明 |
|------|------|------|
| 流式渲染帧率 | ≥ 60fps | 使用 requestAnimationFrame 批处理，避免逐 chunk DOM 重建 |
| DOM 更新频率 | 每帧最多 1 次 | SSE chunk 累积到 rAF 回调中统一渲染 |
| 滚动流畅度 | 无抖动 | 智能滚动（用户上滚时不强制拉回）+ scroll-behavior: smooth |
| 停止响应 | < 100ms | AbortController 即时中断流式请求 |
| Markdown 渲染 | 支持完整 GFM | 使用 marked.js，流式时自动处理未闭合代码围栏 |
| 长文本（>5000字） | 无明显卡顿 | 渲染节流 + rAF 批处理 |

## 安全要求

- **API Key管理**：通过环境变量管理，不在代码中硬编码
- **输入验证**：防止注入攻击
- **CORS配置**：允许跨域访问
- **敏感信息**：不在日志中输出API Key

## 可用性要求

- **健康检查**：提供 `/health` 接口
- **错误处理**：优雅的错误处理和用户提示
- **日志记录**：详细的执行日志
- **用户界面**：友好的Web交互界面

## 可维护性要求

- **代码规范**：遵循Python PEP8规范
- **文档完整**：Spec文档分层管理（Me2AI/AI2AI）
- **模块化**：配置、Agent、工具、记忆、提示词分离
- **技能扩展**：支持通过YAML配置新增技能

## 兼容性要求

- **Python版本**：3.10+
- **浏览器**：Chrome、Firefox、Edge 最新版
- **API格式**：OpenAI兼容接口

## 部署要求

- **环境配置**：通过 `.env` 文件管理
- **依赖管理**：`requirements.txt`
- **启动方式**：支持脚本启动和直接运行
