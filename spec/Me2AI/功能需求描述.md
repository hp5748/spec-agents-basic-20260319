# 功能需求描述

## 项目目标

开发一个**智能客服 Agent Demo**，作为智能体开发项目的最小可运行原型，用于验证核心技术路线和功能可行性。

## 目标用户

- 电商平台客服系统的开发者和决策者
- 需要自动化客服解决方案的企业
- AI Agent 技术研究者

## 核心需求

1. **智能对话**：能够理解用户意图并给出准确回复
2. **技能系统**：基于 Skills 的高级能力抽象，支持 SQLite 查询等
3. **对话记忆**：支持多轮对话，记住上下文
4. **流式响应**：实时输出回复内容

## 期望的用户场景

### 场景1：SQLite 查询（核心验证场景）

```
用户：查询张三的信息
Agent：[意图识别] → [Skill匹配: sqlite-query-skill]
       → [执行 executor.py]
       → 找到 1 条记录：
          张三 - 技术部 - 工程师
          邮箱: zhangsan@example.com
```

### 场景2：通用对话

```
用户：今天天气怎么样？
Agent：[意图识别] → [无 Skill 匹配]
       → [LLM 直接响应]
       → 抱歉，我无法获取实时天气信息...
```

### 场景3：多轮对话

```
用户：查询技术部的员工
Agent：找到 3 条记录：张三、李四、王五...
用户：张三的电话是多少？
Agent：[上下文关联] 张三的电话是 138-xxxx-xxxx
```

## 验收标准

### 已满足
- ✅ LLM API 可正常调用并返回结果
- ✅ 服务可正常启动和运行
- ✅ 支持流式响应（SSE长连接）
- ✅ Web 聊天界面可正常使用
- ✅ Skill 元数据可正常加载

### 待满足（关键）
- ⏳ 能够根据用户输入识别意图并匹配 Skill
- ⏳ 能够调用 Skill 的 executor.py 执行逻辑
- ⏳ 能够将 Skill 执行结果返回给用户
