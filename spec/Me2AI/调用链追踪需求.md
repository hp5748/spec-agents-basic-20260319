# 调用链追踪功能需求

## 需求概述

为系统添加调用链追踪功能，在每次对话结尾显示响应来源，让用户清楚了解请求是如何被处理的。

## 业务背景

当前系统支持三种处理方式：
1. **Skill** - 意图匹配后执行特定技能
2. **SubAgent** - 多 Agent 协作处理（计划中）
3. **MCP** - 调用外部 MCP Server 工具（计划中）
4. **LLM** - 直接语言模型回答

当用户提问时，需要明确告知用户响应是由哪个模块产生的。

## 功能需求

### F1: 调用链标识显示

**优先级**: P0

**描述**: 在每次响应结尾添加调用来源标识

**显示格式**:
```
[Skill: sqlite-query-skill]     # Skill 处理
[SubAgent: code-analyzer]        # SubAgent 处理
[MCP: filesystem]                # MCP 工具处理
[LLM]                            # 直接 LLM 回答
[Skill: xxx → SubAgent: yyy]     # 复杂调用链
```

**规则**:
- 所有响应都必须显示来源标识
- 标识显示在响应内容之后，另起一行
- 使用简洁格式，不包含 "调用来源:" 前缀

### F2: 嵌套调用链支持

**优先级**: P1

**描述**: 支持跨模块的复杂调用链追踪

**场景示例**:
```
用户: "分析这段代码的复杂度"

响应: ...代码分析结果...

[Skill: code-analyzer → SubAgent: complexity-calculator]
```

**规则**:
- 使用 `→` 连接多个调用节点
- 按调用顺序排列
- 最多显示 3 个节点，超过则省略

### F3: 流式响应支持

**优先级**: P0

**描述**: 流式响应也需要在结束时显示调用链

**实现方式**:
- 在 SSE 流的 `done` 事件前发送调用链信息
- 或将调用链作为最后一块 content 发送

### F4: 非流式响应支持

**优先级**: P0

**描述**: 非流式 API 响应中也包含调用链信息

**响应格式**:
```json
{
  "session_id": "xxx",
  "response": "回答内容\n\n[Skill: sqlite-query-skill]",
  "chain_info": {
    "source_type": "skill",
    "source_name": "sqlite-query-skill",
    "confidence": 0.95
  }
}
```

## 非功能需求

### N1: 性能要求

- 调用链追踪不应影响响应速度
- 追踪开销 < 10ms

### N2: 兼容性要求

- 不影响现有 API 接口
- 标识格式兼容前端解析

### N3: 可扩展性要求

- 支持未来新增模块类型
- 支持自定义显示格式

## 验收标准

### V1: Skill 调用场景

```bash
# 输入
curl -X POST http://localhost:8000/api/chat/message \
  -d '{"message": "查询所有用户"}'

# 期望输出包含
找到 5 条用户记录...

[Skill: sqlite-query-skill]
```

### V2: LLM 直接回答场景

```bash
# 输入
curl -X POST http://localhost:8000/api/chat/message \
  -d '{"message": "你好"}'

# 期望输出包含
你好！有什么可以帮助你的吗？

[LLM]
```

### V3: 流式响应场景

```bash
# SSE 流应该包含
data: {"type": "content", "content": "回答内容"}

data: {"type": "content", "content": "\n\n[Skill: xxx]"}

data: {"type": "done"}
```

## 技术约束

1. **数据结构**: 使用 dataclass 定义调用链信息
2. **位置**: 追踪器位于 `src/agent/chain_tracker.py`
3. **集成点**: StreamAgent 负责添加调用链标识

## 依赖模块

- `StreamAgent` - 主路由逻辑
- `SkillExecutor` - Skill 执行器
- `SubAgentOrchestrator` - SubAgent 编排器（计划中）
- `MCPClient` - MCP 客户端（计划中）
